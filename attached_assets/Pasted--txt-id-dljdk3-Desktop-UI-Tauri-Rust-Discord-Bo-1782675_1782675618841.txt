# ทำไมต่อกันตรง ๆ ไม่ได้



เพราะอันเก่าเป็น Tauri app ที่รันบนเครื่องผู้ใช้ มี frontend เรียก backend ผ่าน Tauri `invoke()` ไม่ใช่ HTTP API ธรรมดา



ในโปรเจกเดิม `src/api/tauri.ts` เรียกคำสั่งแบบนี้:



```ts id="m8r81j"

invoke('...')

```



เช่น auth, quest, simulator, debug, CDP ฯลฯ



แต่ Discord bot hosting ต้องการระบบแบบนี้:



```txt id="mtdkqf"

node src/index.js

รับ event จาก Discord

อ่าน/เขียน database

ตอบ slash command

```



คนละโลกกันเลย

# มี 3 ทางเลือก

ง่ายสุด เหมาะเริ่มงาน



```txt id="epc96n"

Desktop App เดิม = เก็บไว้

Discord Bot ใหม่ = ทำแยก

บอททำหน้าที่:



```txt id="v33t2x"

/quest-add

/quest-list

/quest-done

/quest-status

/remind

```



อันนี้เอาขึ้น Bot Hosting ได้เร็วสุด



ข้อดี:



```txt id="vh97yj"

ทำง่าย

พังยาก

deploy ได้ไว

ไม่ต้องยุ่งกับ Tauri/Rust เดิม

```



ข้อเสีย:



```txt id="5r7ppf"

แอปเดิมกับบอทยังไม่แชร์ข้อมูลกัน

```

## ทางที่ 2: เชื่อมผ่าน Database กลาง



อันนี้เริ่มเป็นระบบจริง



```txt id="f2y5f2"

Discord Bot

   ↓

Database กลาง เช่น SQLite/Postgres/Supabase/Neon

   ↑

Dashboard หรือ Desktop App

```



ถ้าใช้ database กลาง บอทกับ dashboard จะเห็นข้อมูลเดียวกัน เช่น:



```txt id="we7cb7"

บอทเพิ่ม quest

dashboard เห็น quest

dashboard แก้ status

บอทเห็น status ใหม่

```



ข้อดี:



```txt id="u8ae64"

เชื่อมกันจริง

ข้อมูลไม่แยก

ต่อยอดเป็นเว็บ panel ได้

```



ข้อเสีย:



```txt id="p09mx6"

ต้องมี database

ต้องออกแบบ API/permission

```

## ทางที่ 3: ทำ API Server เป็นตัวกลาง



นี่คือทางที่ดีที่สุดถ้าจะทำจริงจัง



```txt id="b61b88"

bot/

  Discord bot



api/

  Express/Fastify API server



dashboard/

  web dashboard หรือ frontend จากของเก่า

```



โครงจะเป็น:



```txt id="prngk5"

Discord Bot → API → Database

Dashboard → API → Database

```



ตัวอย่าง flow:



```txt id="f7iczu"

/quest-add ใน Discord

        ↓

bot เรียก POST /quests

        ↓

API บันทึกลง database

        ↓

dashboard เปิดมาก็เห็น quest

```

# แล้วอันเก่าจะเอามาใช้ยังไง

## แบบ A: ใช้เป็น reference อย่างเดียว

```



แล้วเขียนบอทใหม่หมด



นี่เร็วสุด



---



## แบบ B: ดึง frontend เก่ามาทำ Web Dashboard



อันนี้คือเอาหน้า UI เดิมมาแปลงเป็นเว็บ dashboard สำหรับบอท



แต่ต้องแก้เยอะ เพราะของเดิมเรียก Tauri `invoke()`

ต้องเปลี่ยนเป็น HTTP API เช่น:



```ts id="6b9dms"

// เดิม

invoke('get_quests_full')



// ใหม่

fetch('https://api.yourdomain.com/quests')

```



สรุปคือต้องแก้ชั้นนี้:



```txt id="2x2lxw"

src/api/tauri.ts

เปลี่ยน/แยกเป็น

src/api/http.ts

```



และค่อยให้หน้า UI เรียก API ใหม่แทน



---



## แบบ C: เก็บ Tauri เป็น Admin App



อันนี้ advanced กว่า



```txt id="ekth2q"

Desktop App เดิม

เปลี่ยนให้เป็น Admin Control Panel

แล้วให้มันยิง API ไปหา bot/database

แต่ผมยังไม่แนะนำตอนนี้ เพราะจะหนักเกินไป

```txt id="dzlvr2"

โครงสุดท้ายที่ควรเป็น:



```txt id="w5euba"

neverdie-quest-system/

  bot/        discord.js bot

  api/        server กลาง

  dashboard/  เว็บจัดการ

  shared/     type/schema ใช้ร่วมกัน

``
